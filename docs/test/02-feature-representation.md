# 테스트 문서: Feature 표현 (인수 테스트 + 종단 의존성)

## 검증 대상 AC
- AC2.1: 로직 코드로부터 인수 기준 도출 (PRD: Feature 표현)
- AC2.2: 테스트 코드로부터 인수 기준 보강 (PRD: Feature 표현)
- AC2.3: 최종 사용자 관점의 인수 테스트 문서 생성 (PRD: Feature 표현)
- AC2.4: feature 단위 종단 의존성 추출 (PRD: Feature 표현)
- AC2.5: 의존성을 별도 구조화 데이터로 저장 (PRD: Feature 표현)
- AC2.6: 코드 변경 시 표현·의존성의 재계산 가능성 (PRD: Feature 표현)

## 테스트 시나리오

### 시나리오 1: 단순 CRUD feature의 인수 기준 도출
- **사전 조건**: "할 일 추가" 같은 단순 엔드포인트 1개로 구성된 feature가 확정됨.
- **실행 단계**: 해당 feature 선택 → "표현 생성" 트리거
- **기대 결과**: 인수 기준이 "주어진 ~ / ~ 했을 때 / ~ 해야 한다" 형식으로 최소 1개 이상 생성된다. 각 인수 기준에 핸들러 코드 위치 링크가 포함된다.
- **검증 AC**: AC2.1, AC2.3

### 시나리오 2: 테스트가 있는 feature의 시나리오 보강
- **사전 조건**: 시나리오 1의 feature에 단위 테스트와 통합 테스트가 존재.
- **실행 단계**: AC2.2 보강 단계 실행
- **기대 결과**: 보강 후의 시나리오 수가 보강 전보다 같거나 많다. 에러 케이스/경계 조건이 추가된 시나리오에는 근거 테스트 위치가 첨부된다.
- **검증 AC**: AC2.2, AC2.3

### 시나리오 3: 로직과 테스트가 모순되는 경우
- **사전 조건**: 코드 로직은 입력 길이 제한 100자, 테스트는 50자 한계를 가정.
- **실행 단계**: 표현 생성 → 모순 검출 결과 화면 확인
- **기대 결과**: "코드와 테스트가 다르게 말하는 부분" 섹션에 해당 차이가 표시되며, 사용자는 어느 쪽을 정설로 채택할지 결정할 수 있다.
- **검증 AC**: AC2.2

### 시나리오 4: 비개발자 검증 가능성
- **사전 조건**: 시나리오 1, 2 완료된 feature.
- **실행 단계**: 비개발자(예: PM)에게 인수 테스트 문서를 보여주고 그 feature가 무엇을 하는지 설명하게 한다.
- **기대 결과**: 비개발자가 코드 없이도 기능을 정확히 설명할 수 있다.
- **검증 AC**: AC2.3

### 시나리오 5: 단일 feature의 종단 의존성 추출
- **사전 조건**: "결제 완료" 같이 다층 의존성을 가진 feature가 확정됨.
- **실행 단계**: 해당 feature 선택 → "의존성 분석" 트리거
- **기대 결과**: 인프라(예: 결제 PG, DB), 데이터 모델(주문/결제 엔티티), 미들웨어(인증), 프레임워크, 핵심 로직 모듈, 진입점(엔드포인트/UI)이 각 카테고리별로 분리되어 표시된다. 각 항목에 코드 근거가 있다.
- **검증 AC**: AC2.4

### 시나리오 6: 의존성 데이터의 역방향 질의
- **사전 조건**: 다수 feature에 대해 의존성 추출 완료.
- **실행 단계**: "데이터 모델 `Order`를 사용하는 feature 전부" 질의 실행
- **기대 결과**: 해당 모델을 의존성으로 갖는 feature 목록이 반환된다. "미들웨어 인증을 거치는 feature 전부" 같은 다른 차원의 질의도 동일하게 동작.
- **검증 AC**: AC2.5

### 시나리오 7: 의존성 데이터 export
- **사전 조건**: 의존성 데이터 보유.
- **실행 단계**: 데이터 export 트리거
- **기대 결과**: 구조화된 형식(JSON/그래프 형식 등)으로 외부에서 재사용 가능한 형태로 내려받을 수 있다.
- **검증 AC**: AC2.5

### 시나리오 8: 코드 변경 후 재분석 diff
- **사전 조건**: 시나리오 5 완료. 이후 코드에서 결제 모듈에 환불 로직 추가.
- **실행 단계**: 재분석 → 해당 feature 표현 갱신
- **기대 결과**: 인수 테스트 시나리오에 환불 케이스가 추가되거나, 의존성에 환불 관련 모듈이 추가된 차이가 diff로 표시된다.
- **검증 AC**: AC2.6

# PRD: 플랫폼 (k8s · 자격증명 · 모바일 우선)

> 요구사항 #11, #12, #13을 묶은 PRD.
> 핵심 기능 PRD(분석/표현/문서 관리) 모두가 동작하기 위한 비기능적 토대를 정의한다.

## 달성 가치
- **V6: 사용자가 통제하는 자격증명·비용** — GitHub App 설치(저장소 접근 범위) / LLM API Key 관리.
- **V7: 모바일에서 즉시 검토·수정 가능** — UX의 기본 전제.
- **V8: 운영 환경에서의 안정적 가용성** — k8s 운영.

## 범위 / 비범위
- **범위**: 자격증명 등록·저장·사용·폐기, 모바일 우선 UX 원칙, k8s 배포·확장성·관측 가능성.
- **비범위**: 사용자 본인 인증/계정 관리(별도 영역), 결제(사용자 LLM 비용은 사용자 키로 직접 발생).

## Acceptance Criteria

### AC4.1: GitHub App 설치를 통한 저장소 접근 연결·변경·해제
- **설명**: 사용자는 FeatureDoc GitHub App을 자신의 계정 또는 조직에 설치하고, 분석을 허용할 저장소를 선택할 수 있다. App은 분석에 필요한 최소 권한(저장소 콘텐츠 읽기·메타데이터 읽기)만 요청하며, 설치 전에 그 권한 범위를 사용자에게 안내한다. 시스템은 장기 사용자 비밀을 보관하지 않고, 호출 시점에 단기 설치 액세스 토큰(installation access token)을 발급해 사용한다. 사용자는 언제든 저장소 선택 범위를 변경하거나 App 설치를 해제할 수 있다.
- **달성 가치**: V6
- **검증 방법**: 발급되는 설치 액세스 토큰은 단기 만료되며 저장소 측에서 평문 형태로 조회 불가. App 설치 해제 또는 저장소 접근 범위 축소 시 진행 중 분석 작업도 명시된 정책에 따라 처리(중단/현재 호출까지만 완료)된다.

### AC4.2: 사용자 LLM API Key 등록·교체·폐기 및 호출 위임
- **설명**: 사용자는 자신의 LLM 제공자 API Key를 등록할 수 있고, 시스템은 모든 LLM 호출을 해당 키로 수행한다. 키는 평문 저장되지 않으며, 사용자는 호출 누적 횟수/추정 비용을 확인할 수 있다.
- **달성 가치**: V6
- **검증 방법**: 키 폐기 시 신규 LLM 호출이 차단됨. 사용자는 분석 작업 시작 전에 예상 호출 규모를 안내받는다.

### AC4.3: 자격증명의 안전한 사용 정책
- **설명**: 자격증명(GitHub App 개인 키, 단기 설치 액세스 토큰, LLM API Key)은 (a) 저장 시 암호화, (b) 메모리상에서도 필요한 호출 직전에만 복호화, (c) 로그/오류 메시지에 출력 금지의 원칙을 따른다. 자격증명을 다루는 코드 경로는 감사 가능하도록 기록된다.
- **달성 가치**: V6, V8
- **검증 방법**: 시스템 로그에 토큰/키가 노출되지 않음을 검증. 키 사용 이력은 사용자가 조회 가능.

### AC4.4: 모바일 우선 UI/UX
- **설명**: 주요 화면(분석 진행 상황, feature 목록, 인수 테스트 검토, LLM 보조 수정, 의존성 조회)은 모바일 폭(예: 360–430px)에서 1차로 설계되며, 한 손 조작과 짧은 세션을 가정한다. 긴 분석 결과는 단계적으로 노출(요약 → 상세)된다.
- **달성 가치**: V7
- **검증 방법**: 주요 작업이 모바일 환경에서 1세션 5분 이내에 완료 가능. 데스크톱은 동일 화면을 확장 적용한다.

### AC4.5: k8s 배포 및 워크로드 분리
- **설명**: 시스템은 k8s에서 동작하며, (a) 사용자 요청을 받는 API 워크로드와 (b) 분석을 수행하는 백그라운드 워커 워크로드를 분리한다. 분석 작업은 큐를 통해 비동기로 실행된다.
- **달성 가치**: V8
- **검증 방법**: 워커가 다운되어도 API는 응답 가능. 워커는 수평 확장 가능.

### AC4.6: 관측 가능성과 비용 가시성
- **설명**: 분석 단계별 소요 시간, LLM 호출 횟수/토큰 사용량, 실패율이 운영자가 볼 수 있는 형태로 기록된다. 사용자에게도 본인 작업 기준의 누적 호출/추정 비용이 노출된다.
- **달성 가치**: V8, V6
- **검증 방법**: 임의 사용자의 임의 분석 작업의 단계별 비용을 운영자/사용자 모두가 추적 가능.

### AC4.7: 분석 작업의 격리
- **설명**: 한 사용자의 자격증명/저장소/분석 결과는 다른 사용자의 작업에서 접근 불가. 분석 워커는 사용자 단위로 격리되어 실행된다.
- **달성 가치**: V6, V8
- **검증 방법**: 권한 우회 시나리오 테스트에서 다른 사용자 데이터 접근 불가 확인.

## AC 간 관계
- AC4.1~4.3은 자격증명 영역(V6).
- AC4.4는 UX 영역(V7).
- AC4.5~4.7은 운영 영역(V8).
- AC4.7은 V6 + V8 양쪽에 걸친 보안·운영 요구.

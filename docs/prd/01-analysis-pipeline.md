# PRD: 코드베이스 분석 파이프라인

> 요구사항 #5, #6, #7을 한 흐름으로 묶은 PRD.
> "횡단 관심사 분석 → feature 탐색 전략 생성 → feature 목록 추출"의 3단계 파이프라인을 정의한다.

## 달성 가치
- **V1: 코드에서 시작하는 feature 발견 가능성** — feature 발견의 모든 근거를 코드베이스에서 만들어내는 파이프라인 자체.
- **V2: 횡단·종단 양방향 이해** — 1단계가 횡단, 3단계가 종단의 시작.
- **V4: 코드와 일치하는 살아있는 문서** — 분석은 항상 현재 코드 기준으로 다시 실행 가능해야 한다.

## 범위 / 비범위
- **범위**: 저장소 1개를 대상으로 한 1회 분석 파이프라인의 입력·출력·실행 모델.
- **비범위**: 분석 결과를 사용자 관점으로 표현(인수 테스트화)하는 일은 PRD-2에서 다룬다. 의존성 그래프 저장도 PRD-2.

## Acceptance Criteria

### AC1.1: 저장소 연결 및 분석 트리거
- **설명**: 사용자가 GitHub 저장소 URL과 브랜치를 지정하면, 시스템은 해당 저장소를 GitHub App 설치 권한(단기 설치 액세스 토큰)으로 가져와 분석 가능한 상태로 준비한다. 지정한 저장소는 App 설치 시 접근을 허용한 범위에 포함되어 있어야 한다. 사용자는 분석을 명시적으로 트리거할 수 있다.
- **달성 가치**: V1, V6
- **검증 방법**: 모바일 화면에서 repo URL 입력 → 분석 시작 버튼 → 분석 작업이 큐에 등록됨을 확인. App이 설치되지 않았거나 접근 범위에 없는 repo는 명확한 오류 메시지 반환.

### AC1.2: 횡단 관심사 자동 추출 및 문서화
- **설명**: 시스템은 코드베이스로부터 인프라(배포 매니페스트/IaC), 저장소 구조(monorepo 여부, 모듈 구분), 아키텍처(레이어 구분, DDD/MVC/헥사고날 등 패턴), 프레임워크/런타임, 미들웨어(인증/로깅/캐시 등)를 LLM의 도움을 받아 추출하고, 항목별로 정리된 횡단 관심사 문서를 생성한다.
- **달성 가치**: V2, V4
- **검증 방법**: 분석 완료 후 횡단 관심사 문서가 생성되며, 각 항목에는 근거가 된 파일 경로/심볼 참조가 첨부된다. 동일 repo 재분석 시 결과가 결정적으로 재현되거나 차이가 명시된다.

### AC1.3: feature 탐색 전략 생성
- **설명**: 시스템은 AC1.2에서 만든 횡단 관심사 문서를 입력으로 받아, "이 코드베이스에서 최종 사용자용 feature는 어디서 어떻게 발견될 수 있는가"에 대한 탐색 전략을 LLM으로 생성한다. 전략은 사람이 읽고 편집할 수 있는 형태(예: 진입점 파일 패턴, 라우팅 정의 위치, UI 컴포넌트 경로, 외부에 노출된 API 엔드포인트, CLI 명령 등)로 표현된다.
- **달성 가치**: V1, V2
- **검증 방법**: 횡단 분석 결과 변경 시 전략도 재생성될 수 있음. 사용자는 자동 생성된 전략을 검토·수정·승인할 수 있고, 승인된 전략만 다음 단계의 입력이 된다.

### AC1.4: feature 후보 목록 추출
- **설명**: 시스템은 AC1.3에서 승인된 탐색 전략을 따라 코드베이스를 스캔하고, 최종 사용자에게 노출되는 기능 후보 목록을 추출한다. 각 후보에는 발견된 위치(파일·심볼)와 추정 근거가 함께 기록된다.
- **달성 가치**: V1, V2, V4
- **검증 방법**: 추출 결과가 후보 목록(검토 가능)으로 사용자에게 제시되며, 사용자는 후보를 승인/거부/병합/이름 변경할 수 있다. 거부된 후보의 사유는 다음 분석 시 참고될 수 있도록 기록된다.

### AC1.5: 분석 작업의 비동기 실행과 진행 상황 가시성
- **설명**: 모든 분석 단계는 비동기로 실행되며, 사용자는 모바일에서 진행률·현재 단계·예상 LLM 호출 비용/실제 누적 비용을 확인할 수 있다. 실패 시 단계별로 재시도할 수 있다.
- **달성 가치**: V6, V7, V8
- **검증 방법**: 분석 중 앱을 종료해도 진행 상태가 보존되며, 다시 열었을 때 동일한 진행률이 표시된다. 실패한 단계는 그 단계만 재시도 가능하다.

## AC 간 관계
- AC1.1 → AC1.2 → AC1.3 → AC1.4 의 순서로 의존한다.
- AC1.5는 AC1.1~1.4 전체에 걸쳐 적용되는 횡단 요구.
